Project Idea

Overview:

The project I would like to create would be a smart phone application that has the primary function of being an interactive shopping list. Essentially, I am looking a developing an app that will allow groups of people such as friends living together or a family to sign up as a connected private group, where the users of that group can login on their own devices and add items to the shopping list, so at the end of the week there will be an accumulated list of items, from each person if they choose to.

Motivation:

The motivation for developing this project comes from my own personal experiences where I have been constantly met with the frustration of opening the cupboard for food to only find bread or cereal. Which lead me to the idea of setting up an application where everyone in the household could add items to a list when they think of it, rather then forgetting and being met with the same frustration. Also, I see this project to have potential of being successful as it poses a solution to current outdated method of think of items and writing them on a piece of paper before you go to the shops.

Description:

Feature 1 – Create Account

This feature allows users to create their own personal account protected by a password. In order for them to signup the user will need to provide an email address, username and lastly a password followed by 3 security questions. Upon signing up, users will be given access to the features of the app allowing them to begin their experience.

Feature 2 – Group Connection

This feature being the heart and sole of the app, will allow users to setup an account and create a private group that is protected by a 4-digit pin, once the group has been created the user will then be able to invite fellow family members or roommates who have also signed up for the app. In the group their will be a shared document (The shopping list) where each user will have the ability to add items or remove items. When setting up the group, users will be able to choose an icon to identify themselves which will appear next to the items they add to the list, so users know who the groceries are for , avoiding confusion.

Feature 3 – Shopping list

The shopping list is an interactive document which is accessible to all allocated members of the created group (from group connections). The document will have the functionality and capability of having items added and subtracted from the list. Users will also have the ability to add a quantity and if desired add the price of the items. Also, when users are entering an item to the list, they will be given an option of uploading an accompanying image if they desire to. Additionally, in the document settings users will be given an option to choose if the list will refresh once a week (clearing it) or keep the list until everything has been checked of. Also users will be given an option to add a permanent list which will appear at the top of the document every time the list is cleared. Users can use this function to ensure items that will always be needed won’t be forgotten, for example milk, toilet paper etc. Users will also be able to set up restrictions on the document, for example limiting items to save money, or restricting children from adding unhealthy food.

Feature 4 - Store catalogues (updated weekly)

In the menu of the app, users will be able to access a range of catalogues from the big brand shopping centers such as Coles, Woolies and ALDI’S. If users see an item they need or want, they will be able to add the item to their shopping list by easily clicking on the item and choosing a quantity. Once a user has added an item from a store catalogue their own personal catalogue will be created titled “Personal catalogue”, in this catalogue every item they have chosen previously will be stored so each user will have their own personalized catalogue to choose the items they always want or need each week. Additionally, there will also be a group catalogue which is identical to “Personal catalogue” however it includes everyone’s catalogue into one big one, allowing an ease of selection.

Tools and Technologies:

Software –

The software that would be needed to develop this project is XCode, which is an ‘integrated development environment for macOS’ which contains a suite of software development tools. Also, the latest version of MacOS will be required.

Hardware –

A mac computer will be needed to run XCode allowing the development of the app. Also, a phone will be needed to test the application and fix any errors or glitches within the application.

Server –

A server will be needed in order to host the application so that user’s data can be stored and transmitted to multiple devices.

Skills required:

Software –

For the software side of this project the skills that will be necessary include a competent level and understanding of how to use the programming language Swift. Also, I would need to have the ability to use XCode whilst also have the skills to trouble shoot and handle errors.

Feasibility -

To find the skills necessary to complete this project will be relatively easily as there are numerous tutorials online including YouTube guides on how to learn the programming language Swift. Additionally, there are user guides on how to properly use the development environment ‘XCode’. Also, the knowledge of trouble shooting, and error handling will be learnt throughout my course at RMIT.

Outcome:

If the project is successful, the likely outcome would be that the app would be approved onto the app s market place, dragging in thousands of users around the globe to use the interactive shopping list application. Once people start using the application the problem of constantly forgetting groceries will slowly start to fade as the app integrates into their lives making their shopping experiences much simpler and easier for everyone. The impact of this development will add to the long list of presenting to the world how when used correctly, technology can benefit humanity and make usually tedious tasks simple and effortless.
